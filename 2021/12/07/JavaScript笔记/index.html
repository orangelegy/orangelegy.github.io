数据类型转换

1.转换为字符串

var num = 1;
console.log(num.toString());	// 1.toString()
console.log(String(num));		// 2.String()强制转换
console.log(num + "我是字符串")	// 3.加号拼接字符串

2.转换为数字型*

var age = prompt('请输入您的年龄')；	// 获取的值为字符型
// 1.parseInt()转换为整数数值型
console.log(parseInt(age));	
console.log(parseInt('3.14'));		// 返回 3 （不进行四舍五入）	
console.log(parseInt('120px'));		// 返回 120
console.log(parseInt('rem120px'));	// 返回 NaN

// 2.parseFloat()转换为浮点数数值型
console.log(parseFloat('3.14'));		// 返回 3.14
console.log(parseFloat('120px'));		// 返回 120
console.log(parseFloat('rem120px'));	// 返回 NaN

// 3.Number()强制转换
console.log(Number('123'));

// 4.利用算术运算符 - * / 隐式转换
console.log('12' - 0);		// 12
console.log('12' - '10');	// 2
console.log('12' * 1);		// 12

3.转换为布尔值

  • 利用Boolean()强制转换
  • ' ' 0 NaN null undefined 转换为 false
  • 其余值都会被转换为true

tips：利用==进行比较时，默认转换数据类型，会把字符串型数据转换为数字型

短路运算（逻辑中断）

短路运算的原理：当有多个表达式（值）时，左边的表达式值可以确定结果时，就不再继续运算右边的表达式的值。

1.逻辑与

  • 语法：表达式1 && 表达式2
  • 如果表达式1为真，则返回表达式2
  • 如果表达式1为假，则返回表达式1

2.逻辑或

  • 同上

tips：逻辑中断很重要，可能会影响运算结果

  • 函数形参和实参个数不匹配的问题
参数个数 说明
实参个数等于形参个数 输出正确结果
实参个数多余形参个数 只取到形参的个数

tips：return只能返回一个值，如果return多个值，默认返回最后一个值。如果函数中没有return则返回undefined

利用数组可以返回多个值

  • arguments的使用

    当我们不确定有多少个参数传递的时候，可以使用arguments来获取。在Javascript中，arguments实际上它是当前函数的一个内置对象。所有函数都内置了一个arguments对象，arguments对象中存储了传递的所有实参。

    arguments展示形式是一个伪数组

    • 具有length属性
    • 按索引方式储存数据
    • 不具有数组的push，pop等方法

  • 函数表达式的两种声明方式

    1. 利用函数关键字自定义函数
    function fn(){

}
    1. 函数表达式（匿名函数）
    var fun = function(fn){

}

tips：函数的形参实际上就是局部变量

  • 变量的作用域

    • 全局变量：在任何一个地方都可以使用，只有在浏览器关闭时才会被销毁，因此比较占内存
    • 局部变量：只在函数内部使用，当其所在的代码块被执行时，会被初始化；当代码块运行结束后，就会被销毁，因此更节省内存空间。

  • 作用域链

  • 预解析

    • JS引擎运行JS分为两步：预解析 代码执行

      • 预解析JS引擎会把JS里所有的var还有function提升到作用域的最前面
      • 代码执行按照代码的书写顺序从上往下执行

    • 预解析分为变量预解析（变量提升）和函数预解析（函数提升）

      • 变量提升 就是把所有的变量声明提升到当前的作用域最前面，不提升赋值操作
      • 函数提升 就是把所有的函数声明提升到当前作用域的最前面，不调用函数

    • 案例：

      预解析前：
	var num = 10;
	function fn(){
    	console.log(num);
		var num = 20;
    	console.log(num);
	}
	fn();

      预解析后
相当于以下代码：
	var num;
	function fn(){
    	var num;
    	console.log(num);	//输出 undefined
    	num = 20;
    	console.log(num);	//输出20
	}
	num = 10;
	fn();
